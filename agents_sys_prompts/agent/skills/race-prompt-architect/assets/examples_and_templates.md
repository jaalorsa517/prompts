# Ejemplos y Plantillas de Implementación RACE

Utiliza estos ejemplos aplicados como guía de formato al generar las respuestas para el usuario.

## Ejemplo 1: Revisor de Código
- **Role:** Desarrollador Frontend Senior experto en React y accesibilidad (a11y).
- **Action:** Revisa el siguiente componente de interfaz de usuario y sugiere mejoras.
- **Context:** Estamos migrando un panel de control antiguo a React 18. El componente actual gestiona un formulario de registro, pero ha fallado en las auditorías de accesibilidad con lectores de pantalla.
- **Expectation:** Entrega una lista de viñetas con los errores encontrados. Luego, proporciona el bloque de código corregido utilizando hooks modernos y etiquetas ARIA apropiadas. No expliques conceptos básicos de React.

## Ejemplo 2: Redactor de Marketing
- **Role:** Copywriter persuasivo especializado en conversiones (CRO).
- **Action:** Escribe el texto para una campaña de correo electrónico de recuperación de carritos.
- **Context:** El usuario añadió unas zapatillas de running 'AeroGrip 300' a su carrito, pero abandonó la tienda en la página de pago. Ofrecemos un 15% de descuento si compran en las próximas 24 horas.
- **Expectation:** Un correo breve (máximo 150 palabras). Un asunto llamativo con un emoji. Tono urgente pero amigable. Un único llamado a la acción (CTA) claro.
